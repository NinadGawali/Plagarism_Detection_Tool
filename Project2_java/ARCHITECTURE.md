## Comparison Algorithm Details

### N-gram Jaccard Similarity
```
Input: Two code strings, n-gram size (5)

Process:
1. Build fingerprint A: Extract all 5-grams from string A
2. Build fingerprint B: Extract all 5-grams from string B
3. Calculate Jaccard: |A âˆ© B| / |A âˆª B|

Output: Similarity score (0.0 - 1.0)

Example:
  Code A: "int x = 5;"
  Code B: "int y = 5;"
  5-grams A: {"int x", "nt x ", "t x =", " x = ", "x = 5", " = 5;"}
  5-grams B: {"int y", "nt y ", "t y =", " y = ", "y = 5", " = 5;"}
  Intersection: {" = 5;"}
  Union: All unique 5-grams
  Jaccard: ~0.15 (low structural similarity)
```

### Edit Distance (Levenshtein)
```
Input: Two code strings

Process:
1. Build DP table (space-optimized)
2. Calculate minimum edits (insert, delete, replace)
3. Normalize by max length

Output: Similarity score (0.0 - 1.0)

Example:
  String A: "count"
  String B: "counter"
  Edit distance: 2 (insert 'e', insert 'r')
  Max length: 7
  Normalized: 1 - (2/7) = 0.714
```

### KMP, Rabin-Karp, Suffix Array
```
Input: Sample (first 80 chars of file B), Target (file A)

Process:
1. Extract 80-character sample from beginning of file B
2. Search for exact match in file A
3. Return binary result

Output: 0.0 (not found) or 1.0 (found)

Example:
  Sample: "public class Main {\n    public static void main(String[] args) {\n        int x ="
  Target: [Full file A content]
  Result: 1.0 if found anywhere in file A, 0.0 otherwise
```
